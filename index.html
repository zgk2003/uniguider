Introduction

The Linux command line is an interface that allows you to enter text commands to interact with your computer's operating system. Unlike a graphical user interface (GUI) that provides a visual way of interacting with your computer, the command line requires you to enter specific commands to perform tasks. While the command line may seem daunting at first, it offers many advantages, such as more precise control over your system and the ability to perform complex tasks more efficiently (Sobell, 2014).

As a beginner Cornell student, you may need to use the Linux command line in classes such as systems programming and computer architecture. These courses typically require you to work with the command line to perform tasks like compiling code, managing files and directories, and running scripts (Sowell et al., 2010). This guide is designed to help you get started with the Linux command line by covering the basics of using it.

We'll start by showing you how to open the terminal, which is the program that allows you to access the command line. From there, we'll cover basic Linux commands that you'll use frequently, such as navigating the file system, creating and deleting files and directories, and managing processes (Švábenský et al., 2021).

Next, we'll cover file management, including how to copy, move, and rename files and directories. We'll also introduce you to text editors, which are programs used for creating and editing text files. These are all very important skills to master to use Linux in the real working environment (Blank et al., 2015).

Redirection is another powerful feature of the Linux command line that allows you to redirect input and output streams from one command to another (Bandi et al., 2011). We'll show you how to use redirection to perform tasks like saving output to a file or passing input from a file to a command.

Environment variables are variables that store information about the environment in which a program is running, and we'll cover how to use them in the Linux command line (Wescott, 2017).

Finally, we'll cover some common scenarios that utilize multiple skills in order to preform more complex tasks (Jiang & Song, 2015).

In addition, we understand that it might be difficult and potentially dangerous to set up your own terminal environment (Bandi et al., 2011). As such, we prepared an online linux terminal simulator, which could be accessed here. Note that this entire program was created by members of our team from scratch and is not based on any open-source application.

By the end of this guide, you should feel confident in your ability to use the Linux command line and be ready to tackle more advanced topics. Whether you're a computer science student or just interested in learning more about Linux, mastering the command line is an essential skill that will serve you well in your academic and professional career (Weiss et al., 2017).

Opening the Terminal

The terminal is a program that acts as an interface between the user and the operating system of a computer. It provides a command line interface where users can enter text-based commands to perform various tasks on their system. The terminal is an essential tool for users of the Linux operating system, as many tasks can only be performed through the command line.

To open the terminal, you can either use the keyboard shortcut Ctrl + Alt + t or search for the terminal application in your computer. The keyboard shortcut is a quick and easy way to open the terminal, and it can save you time if you need to open it frequently.

Once you've opened the terminal, you'll see a prompt where you can enter commands. The prompt usually displays your username, the name of your computer, and the current working directory (the folder you're currently in). The prompt may also display other information, such as the time and date, the version of the operating system you're using, and the name of the terminal application you're using.

Fig. 1. The Linux Terminal Interface.

When you enter a command in the terminal, the system will execute it and provide output, if any. The output may be displayed in the terminal window or redirected to a file or another command. If there is an error or if the command is not recognized, the terminal will display an error message.

If you are remotely connecting to a Linux computer using Secure Shell (SSH), the connection will automatically connect you to a terminal which is then forwarded to your local computer. This allows you to access the command line of the remote system from your local computer, which can be useful for managing remote systems or accessing resources that are not available on your local system.

In summary, the terminal is a powerful tool that provides a command line interface for users to interact with the operating system of their computer. By learning how to use the terminal, users can perform a wide range of tasks and gain a deeper understanding of how their system works.

Basic Linux Commands

Most Linux commands consist of two possible parts:

  • A function or program that you want to run

  • Inputs to that function or program that modify its behavior

The name of the program you want to run is the first part of the command, which is followed by the inputs. Usually each input is separated by a space from the program name and other inputs.

Here's a list of basic Linux commands that you'll use frequently:

  • pwd: Print the current working directory.

  • ls: List the contents of the current directory.

  • ls <directory>: List the contents of the a different directory.

  • cd <folder name>: Change the current directory.

  • mkdir <folder name>: Create a new directory.

  • touch <new file name>: Create a new file.

  • cp <original location> <new copy location>: Copy a file or directory.

  • mv <original location> <new location>: Move a file or directory.

  • rm <file name>: Delete a file.

  • rm -r <directory name>: Delete a directory and all its contents.

  • man <command name>: Display the manual page for a command.

  • exit: Close the terminal.

In our list of examples commands we use <angle brackets> with text inside to describe the purpose of an input to a function. When running these commands yourself you should replace the <angle brackets> with plain text not inside angle brackets. For example, if our example command was mkdir <folder name> then what you would actually enter into a command line would be mkdir my-folder

Examples:

xxxxxxxxxx
$ pwd
/home/user
$ ls
Documents  Downloads  Music  Pictures  Videos
$ cd Documents
$ mkdir my_project
$ cd my_project
$ touch example.txt
$ cp example.txt example_copy.txt
$ mv example_copy.txt Documents/
$ rm example.txt
$ man ls
$ exit`

File Management

Files in Linux exist inside a directory which is also commonly refered to as a folder. The terminal behaves sort of like a file manager application, like Windows Explorer or Finder, letting you move, copy or delete files and folders. A terminal usually starts inside your home folder which has subdirectories like Documents or Download. You can navigate into folders using the cd command followed by the folders name. If you want to move multiple subfolders at once, you can seperate the name of each subfolder with the character /, the folder name Documents/School refers to the school subfolder of the documents folder.

Unlike the GUI file manager applications you can't press the back button to navigate back to the previous folder. Instead Linux uses the folder name .. to refer to the folder that contains the current folder. So to navigate back from the Documents folder to the home folder you would run the command cd ..

Each file or directory has are two different ways to be named in the command line, reletive and absolute. Absolute file names beign at the root directory / or your home directry ~/ and continue through different subfolders until the file is located. Since there can be many subdirectores the filename can become very long. Instead you can access files using the reletive naming scheme.

Reletive file names are file names that begin with any character besides / or ~/. These file names begin at the directory currently open in the terminal which is called the current directory. The current directory of a Linux terminal can be accessed using the pwd command. Then you can navigate through up and down through the folders to access your desired file.

Examples If a text file called example.txt exists in the folder Documents which is a subfolder of your home folder. Then you can access it from a terminal that running in your home folder using the absolute name ~/Documents/example.txt or the reletive name Documents/example.txt However, if your terminal was located inside the Downloads folder, another subfolder of your home folder, you could access the file using the same absolute name as before ~/Documents/example.txt or the reletive name ../Documents/example.txt

  • rm -r <directory name>: Delete a directory and all its

  • head <file name>: Display the first few lines of a file.

  • tail <file name>: Display the last few lines of a file.

  • diff <file name> <file name>: Compare the contents of two files and display the differences.

Text Editors

When working with the command line, you'll often need to edit files. There are many text editors available for Linux, but we'll focus on two popular command-line editors: nano and vim.

  • nano: A beginner-friendly text editor. It displays a list of keyboard shortcuts at the bottom of the screen. To save and exit, press Ctrl + X, then Y, and finally Enter.

  • vim: A powerful text editor with many features. It has a steeper learning curve than nano, but it's well worth the investment. To enter insert mode, press i. To exit insert mode and return to command mode, press Esc. To save and exit, type :wq and press Enter.

Examples:

xxxxxxxxxx
$ nano example.txt 
$ vim example.txt

Pipes and Redirection

In this section, we will learn about pipes and redirection, the two most powerful features of the Linux command line.

Pipes

A pipe (|) allows you to send the output of one command to another command as input. This is particularly useful when you want to manipulate the output of one command before using it.

For example, let's say you want to search for a specific word in a file. You can use the grep command along with a pipe. First, use the cat command to display the contents of the file, then pipe the output to the grep command:

xxxxxxxxxx
$ cat file.txt | grep "search_term"

This will display all the lines containing the search term in the file.txt file.

Pipes are extremely powerful because they allow simpler programs to be used in complex ways while still remaining simple to use in isolation.
Common Comands to pipe data into:

  • grep: Search for a pattern in the input.

  • wc: Count the number of words from the input.

  • wc -l: Count the number of new lines from the input.

  • awk: Run an AWK script which is a scripting language specialized in manipulating text processing.

  • sed: Finds and replaces text that matches a specific pattern.

Redirection

Redirection allows you to redirect the output of a command to a file or another command. There are three main types of redirection:

  1. Standard output redirection (>): Redirect the standard output (stdout) of a command to a file. If the file does not exist, it will be created. If the file already exists, its contents will be overwritten.

xxxxxxxxxx
$ echo "Hello, World!" > hello.txt

  1. Standard output appending (>>): Similar to the standard output redirection, but appends the output to the file instead of overwriting it.

xxxxxxxxxx
$ echo  "Another line" >> hello.txt

  1. Standard error redirection (2>): Redirect the standard error (stderr) output of a command to a file. This is useful for capturing error messages.

xxxxxxxxxx
$ command_that_generates_errors 2> error_log.txt

You can also combine redirections, for example, to redirect both stdout and stderr to different files:

xxxxxxxxxx
$ command 1> output.txt 2> error_log.txt

Environment Variables

Environment variables are a set of dynamic variables that can affect the way processes run on a computer. They are often used to store configuration settings, paths, and other information that is shared between different programs.

To display the value of an environment variable, use the echo command with the variable name preceded by a $:

xxxxxxxxxx
$ echo  $PATH

To set the value of an environment variable, use the export command:

xxxxxxxxxx
$ export MY_VARIABLE="my_value"

Keep in mind that setting an environment variable using the export command will only affect the current session. To make a variable persistent across sessions, you will need to add the export command to your shell's configuration file (e.g., ~/.bashrc for the Bash shell).

Scenarios

To help you understand how the commands that you just learnt from above could help you with your workflow, we proposed the following three hypotheical scenarios and have provided the corresponding Linux command line solutions to them.

Creating Workspace and File

You are creating a workspace for a new project and want to start work on a program in that area. Make a new folder at ~/work/project and navigate to that folder from the home folder, then create a new file in that folder called project.c

xxxxxxxxxx
# Create the folder '~/work/project' if it doesn't exist and navigate to it
$ mkdir -p ~/work/project && cd ~/work/project
# Create a new file named 'project.c' inside '~/work/project' folder
$ touch project.c

Manipulating Environment Variables

You are writing a program that uses a user’s environment variables to store data. To test this program you will need to modify these variables to different possible starting states. Create a new environment variable called PROGRAM_DATA and store the value 10 in the variable. Then save the current environment variables by storing them in a text file called current_enviroment so you can reuse the current state for future tests.

xxxxxxxxxx
# Create a new environment variable called 'PROGRAM_DATA' and store the value 10 in it
$ export PROGRAM_DATA=10
# Save the current environment variables to a text file called 'current_environment.txt'
$ env > current_environment.txt

Error Handling

You are running a program that is generating hundreds of errors and want to save those errors to a log file. How would you modify the command “program_name -t /etc/” to redirect all the errors to the logfile error.log in your current directory. Then display that file on the terminal.

xxxxxxxxxx
# Run 'program_name' with option '-t' and '/etc/' as an argument, and redirect all errors to 'error.log' file in the current directory
program_name -t /etc/ 2> error.log
# Display the contents of 'error.log' file on the terminal
cat error.log
